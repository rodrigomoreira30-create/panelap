# Markdown.cloud

## Diretrizes Base do Projeto

A partir desta estrutura, todo o desenvolvimento, organização de memória e consulta técnica do projeto deve seguir os padrões abaixo.

---

# 1. Biblioteca e Documentação Técnica

## Regra Principal

Toda biblioteca, framework, SDK, API ou documentação técnica utilizada no projeto deve obrigatoriamente ser consultada através do plugin:

### Context7

## Objetivo

Garantir:

* Padronização técnica
* Consulta de documentação atualizada
* Redução de erros de implementação
* Melhor rastreabilidade de fontes técnicas
* Maior consistência entre agentes, desenvolvedores e automações

## Aplicações

O Context7 deve ser utilizado para:

* Consulta de documentação oficial
* Estrutura de bibliotecas
* Exemplos de implementação
* Boas práticas
* Atualizações de versão
* Compatibilidade entre ferramentas
* Referências técnicas

---

# 2. Memória e Contexto Persistente

## Regra Principal

Toda gestão de memória, contexto persistente e armazenamento contextual do projeto deve utilizar:

### CloudMem

## Objetivo

Garantir:

* Persistência contextual
* Continuidade entre sessões
* Organização de memória operacional
* Histórico de decisões
* Estruturação de conhecimento do projeto
* Memória compartilhada entre agentes

## Aplicações

O CloudMem será responsável por:

* Histórico de interações
* Memória operacional
* Registro de decisões
* Contexto de desenvolvimento
* Preferências do projeto
* Estrutura organizacional
* Evolução do sistema
* Relacionamento entre módulos

---

# 3. Desenvolvimento e Execução Técnica

## Regra Principal

Todo desenvolvimento técnico, automação, execução operacional e construção do projeto deve utilizar:

### Superpowers

## Objetivo

Garantir:

* Padronização de desenvolvimento
* Maior produtividade
* Organização operacional
* Execução avançada de tarefas
* Escalabilidade do projeto
* Estrutura modular de construção

## Aplicações

O Superpowers deve ser utilizado para:

* Desenvolvimento de features
* Automações
* Execução de processos
* Estruturação de workflows
* Organização operacional
* Integrações
* Criação de agentes
* Execução de tarefas técnicas
* Desenvolvimento de módulos

---

# 4. Estrutura Padrão de Funcionamento

## Fluxo Oficial

### Consulta Técnica

Context7
↓

### Persistência e Memória

CloudMem
↓

### Desenvolvimento e Execução

Superpowers

---

# 5. Regras Gerais do Projeto

## Obrigatório

* Toda implementação deve seguir esta estrutura
* Nenhuma biblioteca deve ser utilizada sem consulta prévia ao Context7
* Toda memória relevante deve ser registrada no CloudMem
* Todo desenvolvimento deve seguir o fluxo operacional do Superpowers

## Objetivo Estratégico

Criar uma estrutura:

* Escalável
* Organizada
* Modular
* Inteligente
* Persistente
* Padronizada
* Fácil de expandir
* Fácil de manter

---

# 6. Filosofia do Projeto

O projeto deve ser construído com foco em:

* Organização
* Clareza
* Escalabilidade
* Inteligência contextual
* Automação
* Modularidade
* Continuidade operacional
* Baixa dependência manual
* Facilidade de manutenção
* Estrutura preparada para crescimento
